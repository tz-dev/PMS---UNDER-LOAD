# PMS — UNDER LOAD

*A Structural Self-Critique of the Praxeological Meta-Structure*  
**On Calibration, Coverage, Stack Drift, Publicness, and Self-Application**

---

## Overview

**PMS — UNDER LOAD** is a structural self-critique of the **Praxeological Meta-Structure (PMS)**.

It does not replace PMS Base.  
It does not introduce new PMS operators.  
It does not claim that PMS is false, final, complete, or proven.

Its guiding question is:

```text
Where does PMS stop distinguishing — and start stabilizing?
```

The project tests PMS under the conditions that make it powerful:

```text
calibration
coverage
stack drift
Φ drift
meta-normativity
success trap
domain drift
bridge stress
publicness
self-application
implementation pressure
```

The reduced result is:

```text
PMS survives Under Load not as final grammar,
but as a highly productive, bounded, self-limiting grammar of praxis legibility.
```

## What this project is — and is not

| PMS — UNDER LOAD is | PMS — UNDER LOAD is not |
|---|---|
| a structural self-critique of PMS | a replacement for PMS Base |
| a stress test of PMS’ strongest claims | a rejection of PMS as useless |
| a method for reducing PMS overclaim | a proof that PMS is false |
| a source-discipline and claim-typing framework | a new operator grammar |
| a critique of PMS’ success conditions | a triumphant validation of PMS |
| a way to preserve non-capture and rival pressure | a closure machine |

---

## Core Reductions

```text
universality        → wide portability / generativity
foundation          → operator grammar
completeness        → bounded / versioned / internally constrained
success             → not finality
translation         → not privilege
implementation      → bounded, claim-typed, artifact-supported feasibility where documented
governance          → not tribunal
MIP/AH              → downstream governance / hardening, not rescue layer
QC                  → bridge mapping, not proof
STACK               → implementation pressure, not validation
PMS-RUST            → implementation / artifact evidence, not PMS Base validation
self-application    → not final self-arbitration
rival models        → remain open
non-capture         → remains possible
```

The governing rule is:

```text
A claim becomes unstable when it borrows authority
from a layer other than the one in which it is made.
```

---

## PMS-RUST and Implementation Boundary

PMS now has a documented implementation artifact in the broader ecosystem: **PMS-RUST**.

That matters, but only under reduction.

```text
Implementation is not proof.
Executable artifact-backed partial realization changes the implementation claim,
not the truth-status of PMS Base.
```

PMS-RUST may count as implementation / artifact evidence.

It may not count as PMS Base validation.
